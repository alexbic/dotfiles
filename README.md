# dotfiles

Личные dotfiles (`.zshrc`, `.vimrc`, `.tmux.conf.local`). Клонируются автоматически скриптом `init-shell` / `init-shell-private` при настройке нового сервера.

## claude() — авто-обёртка в tmux (.zshrc)

Функция `claude()` в `.zshrc` переопределяет одноимённую команду:

- **Внутри tmux** (`$TMUX` задан) — просто запускает `claude` как обычно (`command claude "$@"`), без вложенной сессии.
- **Вне tmux** — берёт имя текущей директории (`basename $PWD`, санитизировано под tmux: всё, кроме `a-zA-Z0-9_-`, заменяется на `-`) как имя tmux-сессии:
  - если сессия с таким именем уже существует — подключается к ней (`tmux attach`), продолжая предыдущую работу;
  - если нет — создаёт новую сессию с этим именем и сразу запускает в ней `claude`.

Смысл: сессии tmux переживают закрытие терминала (пока сервер не перезагружен) — можно вернуться в ту же папку и продолжить работу с того же места, а не терять контекст Claude Code.

**Ограничение:** имя сессии — просто имя папки, без пути. Если два разных проекта в разных местах называются одинаково, они попадут в одну и ту же tmux-сессию.
